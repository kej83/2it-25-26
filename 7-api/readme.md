# Om API
En **API** (Application Programming Interface) fungerer som en digital **mellommann** eller "oversetter" som gjør at to forskjellige dataprogrammer kan snakke sammen.

Tenk på det som en **servitør** på en restaurant:

1. **Du (Kunden/Appen)** ser på menyen og bestiller mat.
2. **Servitøren (API-en)** tar bestillingen din med til kjøkkenet.
3. **Kjøkkenet (Serveren)** lager maten basert på bestillingen.
4. **Servitøren (API-en)** kommer tilbake med maten til bordet ditt.

Uten servitøren måtte du selv ha gått inn på kjøkkenet, funnet ingrediensene og laget maten – noe som er både komplisert og usikkert.

---

### Hvorfor bruker vi API-er?

* **Gjenbruk:** Du slipper å bygge alt fra bunnen (f.eks. kart eller betalingsløsninger).
* **Sikkerhet:** API-en deler bare akkurat den informasjonen som trengs, uten å gi tilgang til hele databasen.
* **Automatisering:** Programmer kan hente oppdatert data (som aksjekurser eller vær) automatisk hele døgnet.

---

### Eksempler på anvendelse

Her er noen vanlige situasjoner hvor du bruker API-er hver dag:

| Eksempel | Hva gjør API-en? |
| --- | --- |
| **Vippps / Betaling** | Nettbutikken bruker et API for å spørre Vipps: "Har brukeren betalt?" og få et "Ja" eller "Nei" tilbake. |
| **Logg inn med Google/Facebook** | En nettside bruker et API for å bekrefte identiteten din hos Google, slik at du slipper å lage nytt passord. |
| **Kart i apper** | En treningsapp (som Strava) bruker Google Maps sitt API for å tegne opp ruten din på et kart. |
| **Reiseplanleggere (Entur)** | Appen henter sanntidsdata fra både Vy, Ruter og andre selskaper via deres API-er for å vise deg hele reisen. |
